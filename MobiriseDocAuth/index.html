KBC document authenticity

bootstrap buttons

In times of fake news, also KBC is struggling with incorrect news, influencing their capital shares and public appearance. Documents are shared on the internet, and publiced without the source being checked.

Blockchain technology ables you to easily check the authenticity of all kinds of documents without one party being in control or able to tamper the documents. Cegeka and KBC build an application to verify KBC documents like press releases and presentations. The application offers you the opportunity to check authenticity, and fights fake news. 

Upload original document

Initially, KBC uploads a document (e.g. a press release) in our application. The hash of the document is calculated and stored on the blockchain. For each document, a validity date is added by the KBC officer.

Verify document    

Later on, the visitor of the website who wants to check a certain document uploads his file. The hash is calculated and searched for on the blockchain. 

Get feedback

When the hash is found, the validity date for that document is being checked and a message returns if the document is valid or not. If the document (hash) is not found, the returning message says the document is not known.

Why blockchain?
  1. Immutability - Blockchain allows us the have one single source of truth. The original document â€“represented by the document hash - cannot be tampered.
  2. Proof of existence - At all times, we can proof the original document was stored on that exact date when it was added to the blockchain.
  3. Proof of validity - The smart contract that finds the hash, checks the validity of the document by using the validity date.

Technologies

Ethereum-based project, built on top of our in-house library. Consists of two websites (one public + one internal) that handles proofs of authenticity for documents.

  • Ethereum blockchain
  • Angular 1 frontend
  • Java Spring boot + custom Cegeka Platform library for Ethereum interaction
  • Docker-compose based deployment (PostgreSQL database + NginX frontend webserver + Tomcat Spring Boot backend server + private Ethereum server)
What is a document hash?

The document hash is a unique identifier for a certain document. The hash value is calculated on the document. Whatever letter, symbol, space is changed in the original document, the calculated hash will not correspond to the original hash. This way you can prove that the document you have, has not been tampered when the hash corresponds to the hash that was stored on the blockchain.
